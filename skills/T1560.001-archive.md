# T1560.001 — Archive Collected Data: Archive via Utility

## Tool sequence
1. `find /mnt/host -iname '*.rar' -o -iname '*.7z' -o -iname '*.zip' -o -iname '*.tar' 2>/dev/null` — locate archive files.
2. `find /mnt/host/Windows/Prefetch -iname '*.pf' 2>/dev/null | xargs strings 2>/dev/null | grep -i '7z\|rar\|winzip\|winrar'` — archiver execution evidence.
3. `ls -la <archive_path>` — check file size and timestamp relative to data collection window.

## CONFIRMED when
- Archive file found containing collected data (system4.rar, chrome.7z, or similar staging artifact).
- Archive file timestamp aligns with data exfiltration window.
- Archive utility (7z.exe, WinRAR.exe, rar.exe) found in Prefetch with execution timestamp.
- Archive found in staging location (Desktop, Temp, Recycle Bin, or attacker-created directory).

## REFUTED when
- No archive files in non-standard locations after full image search.
- Archives found but all are legitimate system backups (Windows Backup, standard user archives predating incident).

## Case context
On nfury: `system4.rar` and `chrome.7z` found — confirmed data staging for exfiltration.
Timestamp of archive files was within the attack window. Archive tools appeared in Prefetch.
