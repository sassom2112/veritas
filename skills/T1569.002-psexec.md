# T1569.002 — System Services: Service Execution (PsExec)

## Tool sequence
1. `find /mnt/host -iname 'PSEXESVC.EXE' 2>/dev/null` — primary. Binary must be on disk.
2. `find /mnt/host -iname 'psexec*' 2>/dev/null` — catch variants (psexec64.exe, psexec.exe drop).
3. `find /mnt/host/Windows/Prefetch -iname 'PSEXESVC*' 2>/dev/null` — prefetch execution evidence.

## CONFIRMED when
- PSEXESVC.EXE found anywhere on disk (System32, Temp, attacker drop path, or deleted-but-recoverable via Prefetch).
- Any psexec variant binary found.
- Prefetch file `PSEXESVC.EXE-*.pf` found (confirms execution even if binary was deleted).

## REFUTED when
- No PSEXESVC.EXE or psexec variant found after checking System32, all Temp directories, and all user directories.
- No Prefetch entry for PSEXESVC.EXE.
- Do NOT refute based on absence from one directory alone — check the full image.

## Case context
T1569.002 was CONFIRMED on nfury (PSEXESVC.EXE on disk) and REFUTED on tdungan (binary absent despite
campaign sharing nfury IOCs). The same technique produces different verdicts on different hosts — this is
correct Auditor behavior. Do not assume lateral movement means PsExec binary is present everywhere.
