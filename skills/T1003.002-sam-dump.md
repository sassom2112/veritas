# T1003.002 — OS Credential Dumping: SAM

## Tool sequence
1. `windows.hashdump` — primary for memory images. Extracts NTLM hashes directly from memory.
2. `find /mnt/host -iname 'SAM' -o -iname 'SYSTEM' -o -iname 'SECURITY' 2>/dev/null | grep -v '\.(log\|LOG)'` — locate hive files.
3. `strings <memory_image> 2>/dev/null | grep -c 'aad3b435b51404eeaad3b435b51404ee'` — count LM-null hash (blank password marker).

## CONFIRMED when
- `windows.hashdump` returns NTLM hashes for local accounts (Administrator, SRL-Helpdesk, etc.).
- SAM and SYSTEM hive files found in expected location (`/mnt/host/Windows/System32/config/`).
- LM-null hash (`aad3b435b51404eeaad3b435b51404ee`) count > 0 in memory strings.
- Credential harvester binary (HYDRAKATZ.EXE, procdump, comsvcs.dll) in Prefetch or on disk.

## REFUTED when
- No SAM/SYSTEM hives found and no hashdump output after memory analysis.
- Hive files present but no evidence of access (no tool artifacts, no credential harvester in Prefetch).

## Case context
On nfury: hashdump returned 5 accounts including `SRL-Helpdesk` with NTLM hash `4c3f5e9f...`.
On tdungan: same hash matched — HYDRAKATZ.EXE in Prefetch confirmed credential harvesting.
The NTLM hash reuse across hosts is campaign-level confirmation of lateral movement via credential reuse.
