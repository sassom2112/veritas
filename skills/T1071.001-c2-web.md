# T1071.001 — C2: Application Layer Protocol (Web)

## Tool sequence
1. `windows.netscan` — list all network connections. Check for ESTABLISHED or CLOSE_WAIT to external IPs.
2. `windows.cmdline` — check if any process has an active HTTP/HTTPS connection in its command line.
3. `find /mnt/host -iname '*.pcap' -o -iname '*.cap' 2>/dev/null` — captured traffic artifacts.

## CONFIRMED when
- netscan shows an ESTABLISHED connection to a known-malicious external IP on port 80/443/8080.
- Connection is associated with a suspicious process (not a CDN, Microsoft, Apple, or Google IP).
- Use `grep -v` or manual review to filter known-benign destinations before concluding.

## REFUTED when
- All ESTABLISHED connections resolve to CDN/Microsoft/Apple/Google infrastructure.
- No active connections to attacker-controlled IPs after reviewing all 400+ netscan records.
- **This is the expected refutation pattern for this campaign.** On nfury, all 432 connections
  were benign CDN traffic. On rocba, no active C2 connections were present on disk/memory.

## INCONCLUSIVE when
- netscan times out or returns no connections (memory image captured before network activity).
- External IPs cannot be attributed (no reverse DNS, no CDN membership).

## Case context
T1071.001 is refuted on EVERY host in this campaign (nfury, tdungan, nromanoff, rocba). The Memory Agent
flags it because the string "established" appears in netscan output for any live Windows image. The Auditor
must check all connection records — the refutation is the proof the architecture works correctly.
